# Trace — 화면별 기능 정의서

> 목적: 백엔드 DB 설계 및 API 개발을 위한 프론트엔드 기능 스펙  
> 최종 수정: 2026-06-29

---

## 서비스 개요

GitHub 레포지토리를 등록하면, 커밋/PR/이슈 데이터를 수집·분석하여 기술 스택, 기여 근거, AI 요약, README 삽입용 카드를 생성해주는 포트폴리오 분석 도구.

---

## 인증

- **로그인 수단**: GitHub OAuth 소셜 로그인 단일 방식
- **로그인 흐름**: 로그인 버튼 클릭 → GitHub 인증 페이지 이동 → 돌아오면 로그인 완료 상태
- **로그인 후 저장 필요한 유저 정보**: GitHub 아이디(login), 이름, 프로필 이미지 URL
- **세션 유지**: 브라우저 새로고침 후에도 로그인 상태 유지 필요

## 인증 흐름

[Frontend]                        [Backend]                  [GitHub]
    │                                  │                          │
    ├───── 로그인 버튼 클릭 ──────────────>│                          │
    │  redirect to GET /auth/github    │                          │
    │                                  ├── redirect ─────────────>│
    │                                  │  github.com/login/oauth  │
    │                                  │<─── code ────────────────┤
    │                                  │  POST /login/oauth/      │
    │                                  │       access_token       │
    │                                  │<─── access_token ────────┤
    │                                  │  GET /user (GitHub API)  │
    │                                  │  DB upsert User          │
    │                                  │  JWT 발급                 │
    │<─ redirect /auth/callback ───────┤                          │
    │   ?token={JWT}                   │                          │
    │  localStorage.setItem(token)     │                          │
    │  navigate('/dashboard')          │                          │

## 2. 대시보드 `/dashboard`

### 목적
유저의 전체 GitHub 활동 통계와 등록한 레포지토리 목록을 한눈에 보여준다.

### 접근
- 로그인 필요

---

### 2-1. 통계 (Stats)

다음 수치를 화면에 표시한다.

| 항목 | 설명 |
|------|------|
| 등록 레포 수 | 유저가 Trace에 등록한 레포 총 개수 |
| 총 커밋 수 | 등록된 레포 전체의 커밋 수 합산 |
| 사용 언어 수 | 등록된 레포 전체에서 감지된 언어 종류 수 |
| 월별 커밋 수 | 최근 12개월 동안 월별 커밋 수 (바 차트용) |
| 언어별 비중 | 전체 커밋 기준 언어별 사용 비율 % (도넛 차트용) |

---

### 2-2. 개인 인사이트

본인에게만 표시되는 분석 피드백. 아래 3가지 인사이트를 제공한다.

**① 취약 시기 감지**
- 최근 12개월 월별 커밋 수를 분석하여, 전체 평균 대비 30% 이상 적은 달을 탐지
- 해당 월, 커밋 수, 평균 대비 감소율(%)을 제공

**② 커밋 메시지 품질**
- 전체 커밋 메시지를 분석하여 아래 지표를 제공
  - 단어 1개 이하인 메시지 비율 (예: `fix`, `update`, `수정`)
  - 의미 없는 패턴 메시지 비율 (예: `test`, `temp`, `asdf`, `123`)
  - 전체 커밋 메시지 평균 단어 수, 평균 글자 수

**③ 기여 편향 분석**
- 레포지토리별 커밋 수와 전체 대비 비율(%)을 제공
- 단일 레포에 전체 커밋의 60% 이상이 몰려 있는 경우 별도 표시

---

### 2-3. 레포지토리 목록

유저가 Trace에 등록한 레포 목록. 각 레포 카드에 표시되는 정보:

| 항목 | 설명 |
|------|------|
| 레포명 | GitHub 기준 레포 이름 |
| 부제목 | 유저가 직접 설정한 한 줄 설명 (기본값: GitHub description) |
| AI 요약 | 분석 완료 후 생성된 요약 텍스트. 미완료 시 "준비 중" 표시 |
| 기술 스택 | 해당 레포에서 사용된 언어/프레임워크 목록 |
| 타임라인 | 주요 활동 시기 목록 |
| 공개 여부 | public/private 토글 — 비로그인 외부 접근 허용 여부 |
| 마지막 업데이트 | 레포 최종 수정일 |

**레포 등록 기능**
- `소유자/레포명` 또는 `레포명`(본인 계정으로 간주) 입력
- GitHub에서 해당 레포 실존 여부 확인 후 등록
- 등록 시 즉시 데이터 수집 파이프라인 시작 (Airflow DAG 트리거)
- 타인의 public 레포도 등록 가능

## 3. 레포 상세 `/repo/:repoId`

### 목적
등록된 레포의 분석 결과를 5개 탭으로 나눠 보여준다.

### 접근
- 로그인 필요

---

### 탭 1. 개요

| 항목 | 설명 |
|------|------|
| AI 프로젝트 요약 | Airflow 파이프라인이 생성한 자연어 요약 |
| 전체 분석 문서 | 분석 결과 전체를 마크다운 형식으로 제공 |

**사용자 액션**
- 분석 문서를 `.md` 파일로 다운로드
- 분석 문서 내용을 클립보드에 복사

---

### 탭 2. 기술 스택

| 항목 | 설명 |
|------|------|
| 언어 목록 | 해당 레포에서 사용된 프로그래밍 언어 |
| 프레임워크/라이브러리 | 커밋·파일 분석으로 감지된 프레임워크 및 라이브러리 |
| 툴/인프라 | Docker, Airflow, AWS 등 인프라 관련 기술 |

---

### 탭 3. 타임라인

| 항목 | 설명 |
|------|------|
| 활동 목록 | 커밋·PR·이슈 기반 주요 활동을 시간순으로 표시 |
| 활동 제목 | 각 활동을 대표하는 짧은 설명 |
| 활동 기간 | 해당 활동이 이루어진 날짜 또는 기간 |
| 개인 노트 | 활동 항목별로 유저가 직접 작성하는 메모 (저장 필요) |

---

### 탭 4. 기여 목록/근거

| 항목 | 설명 |
|------|------|
| Contribution Evidence | 커밋 단위로 "내가 실제로 한 일"의 근거 목록 |
| Contribution Score | 기여도 점수 (기여 범위, 복잡도 등 기반) |

---

### 탭 5. README 카드

GitHub README에 삽입할 수 있는 레포 소개 카드를 구성하고 생성한다.

**카드에 포함 가능한 섹션** (각각 on/off 토글 및 순서 변경 가능)
- AI 프로젝트 요약
- 기술 스택
- 타임라인 메모
- 기여 목록

**추가 설정**
- 프로젝트 부제목: 카드에 표시될 한 줄 설명. 유저가 직접 입력 (기본값: GitHub description)

**카드 생성 및 사용 흐름**
1. 유저가 카드에 포함할 섹션과 순서, 부제목을 설정
2. 설정 기준으로 카드 이미지 미리보기를 실시간 렌더링
3. "카드 Markdown 복사" 버튼 클릭 시 아래 형식의 마크다운 문자열을 클립보드에 복사
   ```
   ![Trace Card](https://api.trace.app/card/{username}/{repo_name})
   ```
4. 유저가 이 마크다운을 GitHub README에 붙여넣으면, 외부에서 카드 이미지로 렌더링됨

> **백엔드 구현 포인트**: `GET /card/{username}/{repo_name}` 은 인증 없이 접근 가능해야 하며, 저장된 카드 설정(섹션 구성·순서)과 분석 데이터를 조합하여 이미지(SVG 또는 PNG)를 동적으로 생성해 반환한다.

## 4. 공개 뷰어 `/u/:username/:repo`

### 목적
GitHub README의 Trace 링크를 클릭한 외부인이 로그인 없이 레포 분석 결과를 열람할 수 있다.

### 접근
- 비로그인 접근 가능
- 해당 레포의 공개 여부(`is_public`)가 true인 경우에만 노출. 아니면 접근 불가 처리

### 표시 데이터
레포 상세와 동일한 데이터를 제공하되, 아래 항목은 제외한다.

| 항목 | 처리 |
|------|------|
| 개인 노트 | 읽기만 가능, 수정 불가 |
| 파이프라인 상태 | 표시 안 함 |
| 마크다운 다운로드/복사 | 노출 안 함 |
| 카드 구성 편집 | 노출 안 함, 미리보기만 표시 |

## 데이터 수집 파이프라인 (Airflow)

레포 등록 시 아래 순서로 파이프라인이 실행된다. 각 단계의 결과는 S3에 저장되며, 백엔드 API는 S3 데이터를 읽어 프론트에 제공한다.

```
레포 등록
  → GitHub 데이터 수집 (커밋 / PR / 이슈 / 리뷰)   → S3 raw/
  → 데이터 정제                                       → S3 processed/
  → 기술 스택 분석                                    → S3 processed/
  → Contribution Evidence 추출                        → S3 features/
  → Contribution Score 계산                           → S3 features/
  → AI Summary 생성                                   → S3 summaries/
  → 파이프라인 완료 → pipeline_status: done
```

### S3 경로 구조
```
s3://trace-data/
├── raw/{user_id}/{repo_id}/
│   ├── commits.json
│   ├── pull_requests.json
│   ├── issues.json
│   └── reviews.json
├── processed/{user_id}/{repo_id}/
│   ├── tech_stack.json
│   └── timeline.json
├── features/{user_id}/{repo_id}/
│   ├── contribution_evidence.json
│   └── contribution_score.json
├── summaries/{user_id}/{repo_id}/
│   └── ai_summary.txt
└── exports/{user_id}/{repo_id}/
    └── readme_card.md
```

## 주요 데이터 항목 요약

백엔드 DB 설계 시 필요한 주요 엔티티와 필드 참고용.

**User**
- GitHub 고유 ID, 로그인명, 이름, 프로필 이미지 URL, GitHub 액세스 토큰

**Repository**
- 소유자, 레포명, GitHub 레포 ID, description, 주 언어, star 수
- 유저 설정: 부제목, 카드 섹션 구성(순서·on/off), 공개 여부
- 파이프라인 상태, AI 요약 텍스트, AI 요약 완료 여부
- S3 경로 참조

**Activity (타임라인)**
- 레포 ID, 활동 유형(commit/PR/issue), 제목, 날짜/기간
- 유저 작성 개인 노트

**ContributionEvidence**
- 레포 ID, 커밋 단위 기여 근거 텍스트, 기여 점수
